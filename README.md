# My_project2

This repository contains a small Unity prototype project built around a single playable scene and a basic player movement script. It is set up as an early gameplay sandbox rather than a finished game.

## Project Overview

- Unity version: `6000.3.9f1`
- Render pipeline: Universal Render Pipeline (URP)
- Main scene: `Assets/Scenes/Prototype 2.unity`
- Main gameplay script: `Assets/Scripts/PlayerController.cs`

## Current State

The project currently includes:

- A single prototype scene.
- Basic horizontal player movement.
- X-axis movement clamping so the player cannot move outside the allowed range.
- A course asset library with characters, animals, food props, and environment materials.
- Unity package dependencies for URP, Input System, Ads, Purchasing, Navigation, and related editor tooling.

## Controls

The current player controller uses the `Horizontal` input axis, so movement works with the default Unity mappings such as:

- `A / D`
- Left / Right Arrow keys

## Getting Started

1. Open the project in Unity Hub.
2. Use Unity `6000.3.9f1`.
3. Open `Assets/Scenes/Prototype 2.unity`.
4. Press Play in the Editor.

## Project Structure

- `Assets/Scenes` - playable scene files.
- `Assets/Scripts` - gameplay scripts.
- `Assets/Course Library` - imported prototype assets used for the scene.
- `Assets/Settings` - URP and rendering settings.
- `Assets/MobileDependencyResolver` - third-party dependency management package for Unity.

## Notes

- `Library`, `Temp`, and `Logs` are generated by Unity and do not need to be shared as project source.
- The README inside `Assets/MobileDependencyResolver/Editor` belongs to a third-party package and is not the main project documentation.
- This repository currently looks like a learning or prototype workspace, so expect iteration and unfinished systems.
